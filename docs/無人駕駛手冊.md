# TWSE Alpha Lab 無人駕駛手冊

> 目的:沒有任何外援時,一個人維運整套系統。遇到本手冊沒寫的狀況,原則永遠是:
> **先診斷(純讀)、再修復;懷疑數字就回原始資料驗算;一次只改一個變因。**

---

## 0. 系統自動在做什麼(平常不用管)

每天 18:30 launchd 自動執行 daily_pipeline,順序為:

1. 抓 OHLCV(MI_INDEX 主源,失敗自動改用 STOCK_DAY_ALL 備援;都失敗自動登記 pending_backfill)
2. 技術指標 → 每日評分 → ML 評分(帶 --date,防偷看未來)
3. 除權息表更新 → 策略決策 → 模擬成交 → 除息現金入帳 → 淨值快照
4. benchmark 重建(0050 與 00981A,含息)
5. 每日檢討書 → ML 選股檢討 → 主題熱度 → 資料品質檢查

只要 Mac 在 18:30 是開機且清醒的,以上全自動。

## 1. 每週巡檢(建議週五晚,5 分鐘)

在終端機執行:

    cd ~/twse_quant_platform && source .venv/bin/activate

    sqlite3 data/db/quant.db "SELECT 'OHLCV' t, MAX(trade_date) FROM ohlcv_daily UNION ALL SELECT '分數', MAX(score_date) FROM daily_scores UNION ALL SELECT '淨值', MAX(snap_date) FROM equity_curve UNION ALL SELECT 'benchmark', MAX(snap_date) FROM benchmark_daily_equity;"

    sqlite3 data/db/quant.db "SELECT * FROM pending_backfill WHERE resolved_at IS NULL;"

判讀:
- 四個日期都等於最近一個交易日 → 健康,關掉終端機
- pending_backfill 有未解列 → 執行第 4 節 A 的自動補齊
- 某個日期落後但 pending 是空的 → 執行第 4 節 B

## 3. 月底檢視怎麼做(每月最後交易日之後)

打開 localhost:8000/competition,加上這個指令:

    curl -s localhost:8000/api/monthly/race | python3 -m json.tool

依序回答五個問題:

1. 哪些策略的月 alpha_vs_0050 為正?(真正贏過大盤的)
2. 勝率:**累積平倉 ≥ 20 筆**的帳戶勝率才可信;不足 20 筆一律視為「樣本不足」,不下結論
3. PF 大於 99 或小於 0.1 都是樣本太少的假象,直接忽略
4. A7 純 ML 的累積 alpha 是收斂還是惡化?(這決定 8 月要不要做 ml_rank 整合進主引擎)
5. 回撤最大的帳戶是誰、發生在哪幾天?對照當時大盤是不是同步下跌

**鐵則:單月結論不改參數。** 連續兩個月同方向的證據才考慮動規則,而且一次只改一項、改完記錄在 git commit。

## 4. 故障自救

### A. 漏抓某天(最常見)

    python3 -m scripts.process_pending_backfill

自動補 pending 表裡登記的日期。手動指定日期:

    python3 -m scripts.process_pending_backfill --dates 2026-XX-XX

### B. 檢討書/主題熱度/品質檢查某天沒跑(面板日期停滯)

建一個臨時檔 fix_day.py,內容(把日期改成缺的那天):

    from datetime import date
    from backend.services.latest_update import update_theme_trends
    from backend.v4.data_quality import run_data_quality_checks
    d = date(2026, 7, 15)
    update_theme_trends(d)
    run_data_quality_checks(d)

然後 python3 fix_day.py。檢討書缺的話再加:

    from backend.services.daily_review import generate_daily_review
    generate_daily_review(date(2026, 7, 14), date(2026, 7, 15))  # (前一交易日, 缺的那天)

### C. 網頁掛了或 500 錯誤

    lsof -ti:8000 | xargs kill -9
    nohup uvicorn main:app --host 0.0.0.0 --port 8000 > /tmp/uvicorn.log 2>&1 &
    tail -20 /tmp/uvicorn.log

log 裡若見「Out of range float values are not JSON compliant: nan」= 某個 API 回傳 nan,參考 main.py 裡 sanitize_nan 的寫法處理該端點。

### D. TWSE 限流(連續多天抓空、或長任務大量 NO_DATA)

等 1-2 小時再試。長任務被限流就 kill,隔天重跑同一行指令,進度表自動續傳。

### E. 歷史建設(build_history)中斷續跑

    nohup caffeinate -i python3 -m scripts.build_history_2017_2023 --start 2017-01-01 --end 2023-12-31 --sleep 3 > /tmp/hist_rebuild.log 2>&1 &

查進度:

    sqlite3 data/db/quant.db "SELECT COUNT(*), MAX(trade_date) FROM hist_rebuild_progress;"

注意:caffeinate 必須包在指令裡,否則 Mac 睡著任務就停。

### F. launchd 排程確認

    launchctl list | grep twse
    grep "$(date +%Y-%m-%d)" ~/twse_quant_platform/data/logs/launchd_out.log | tail -10

## 5. 已知未解事項(誠實清單)

- **5871 配股(7/29)**:dividend_income 只處理現金股利;配股的股數增加要人工調整 positions,或接受小誤差並記錄
- **迷你單手續費侵蝕**:單筆買入低於 5,000 元時最低手續費 20 元占比過高,尚未過濾(P2)
- **漲跌停鎖死未建模**:漲停買不到、跌停賣不掉的情況,paper engine 目前假設都能成交,A7 的成交假設偏樂觀(P2)
- **Survivorship bias**:universe 只含現存股票,歷史回測前必修(P3)
- **月營收/財報未入特徵**:若未來要加,必須先建 announce_date 對齊,嚴禁用資料所屬期間對齊
